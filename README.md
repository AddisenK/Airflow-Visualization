# Airflow Visualization - Aircraft Aerodynamics

A modern, interactive 3D web application that visualizes airflow (air particles and streamlines) around an aircraft during different stages of flight. Built with React, TypeScript, Three.js, and a futuristic UI.

## ğŸ§  How It Works

### Airflow Simulation

The application uses a simplified aerodynamic model that emphasizes visual clarity over scientific precision:

1. **Particle System**
   - 5,000 particles are continuously generated in front of the aircraft
   - Each particle follows a simplified physics model:
     - Base velocity towards the aircraft (based on airspeed)
     - Wind effects (lateral and vertical disturbances)
     - Angle of attack influence (vertical deflection)
     - Turbulence around wing surfaces (randomized motion)

2. **Aerodynamic Effects**
   - **Lift Vectors**: Visual representation of upward force generated by wings
   - **Pressure Zones**: Simplified visualization of high (red) and low (blue) pressure regions
   - **Turbulence**: Enhanced particle randomization near wing surfaces

3. **Flight Stage Parameters**
   Each flight stage modifies:
   - Airspeed (knots)
   - Angle of attack (degrees)
   - Wind strength (0-1)
   - Particle density (0-1)
   - Particle velocity multiplier (0-1)

### Technical Implementation

- **React Three Fiber**: React renderer for Three.js, providing declarative 3D scene management
- **Custom Shaders**: GPU-accelerated particle rendering with vertex and fragment shaders
- **Zustand**: Lightweight state management for flight parameters and UI state
- **Tailwind CSS**: Utility-first CSS framework for modern styling

## ğŸ”§ Troubleshooting

### Peer Dependency Warnings (ERESOLVE)

If you encounter `npm WARN ERESOLVE overriding peer dependency` warnings during installation, this is typically harmless and won't affect functionality. However, if you want to suppress these warnings:

1. **Option 1**: Use the legacy peer deps flag:
   ```bash
   npm install --legacy-peer-deps
   ```

2. **Option 2**: Enable it in `.npmrc` by uncommenting the `legacy-peer-deps=true` line

3. **Option 3**: Update all packages to their latest compatible versions (already done in this project)

The warnings occur because some packages have peer dependency requirements that may conflict with other packages. The application will still work correctly despite these warnings.
